# 디자인
## 최소 화면 세트
1. **홈/학습 선택**: N5/N4/N3 레벨 선택, 덱(카테고리) 리스트
2. **플래시카드 학습**
    - 앞면: 한자 + 후리가나
    - 뒷면: 뜻 + 예문 + “부수 기반 의미 유추(설명)”
    - “알겠음/애매함/모름” 버튼 (나중에 복습 로직 확장 가능)
3. **퀴즈 시작 화면**: 문제 수(10/20), 난이도(간단히), 범위(덱/레벨)
4. **퀴즈 진행 화면**: 듀오링고처럼 2~3가지 타입만 먼저
5. **결과 화면**: 맞춘 개수, 틀린 단어 리스트, 다시 풀기 버튼
6. **설정/정보**: 앱 정보, 데이터 출처/라이선스(중요), 문의(메일 링크)

  
# 수익창출
## 광고
### **(1) “프로 사용자” 개념(Entitlement)만 미리 설계**

- 예: isPro 하나로 광고 제거/추가기능 잠금해제
    
- 지금은 항상 false여도 됨
    
    → 나중에 IAP 붙일 때 UI/로직 흔들림이 적어져.
    

  

### **(2) 광고 자리(Placement)도 UI 구조만 미리 고려**

- 결과 화면 하단, 덱 리스트 중간 등 “들어갈 위치”만 확보
    
- 나중에 붙일 때 레이아웃 붕괴 방지
    

  

### **(3) 개인정보/동의 흐름 대비**

- 광고 넣는 순간, 사용자 동의(지역에 따라)나 정책 이슈가 생기기 쉬움
    
- MVP 때부터 **설정 화면에 “개인정보/문의/데이터 출처”**는 만들어 두는 걸 추천
